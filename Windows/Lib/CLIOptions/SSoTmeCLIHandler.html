Command Line Arguments

There are 53 command line arguments defined at this time. They are:

-help or -h [bool]

Show help about how to use the SSoT.me CLI

-init [bool]

Initialize the current folder as the root of an SSoT.me project. An Optional parameter of force will create a sub-project.

-install [bool]

Saves the current command into the SSoT.me Project file

-uninstall [bool]

Removes the current command from the SSoT.me Project file

-build or -b,replay,rebuild,pull [bool]

Build any transpilers in the current folder (or children).

-buildAll or -ba,replayall,rebuildAll,pullAll [bool]

Builds all transpilers in the project

-buildLocal or -bl,replaylocal,rebuildLocal,pullLocal [bool]

Build only the transpilers installed in the current folder

-buildOnTrigger or -bot [string]

Builds whenever a trigger is invoked (see readme for URL)

-copilotConnect or -cc [bool]

Use with buildOnTrigger to connect the baseId to the SSoTme Copilot Agent

-debug [bool]

Show debug output

-describe or -d [bool]

Describes the current SSoT.me Project (and all transpilers)

-descibeAll or -da [bool]

Descibe all of the transpiler in the project

-discuss or -ai [bool]

Discuss the project with an AI

-input or -i [List of string]

Input filename or comma separated list of file names

-output or -o [string]

Output filename

-clean or -c [bool]

Clean all transpilers installed in and downstream of this folder

-cleanAll or -ca [bool]

Clean all project transpilers

-cleanLocal or -cl [bool]

Cleans only transpilers installed in the current folder

-purge [bool]

When supplied with clean, runs clean on all orphaned transpilers

-skipClean or -sc [bool]

Don't clean the output before cooking

-localGuide or -lg [bool]

A local guide that will connect to the my.effortlessapi.com Guide

-listSeeds or -lsd [bool]

Lists seeds available to be clones

-cloneSeed or -cs,clone [bool]

Clones a specified seed

-account or -a [string]

The account which the transpiler belongs to

-parameters or -p [List of string]

A list of parameters

-listSettings or -ls [bool]

List of project settings

-addSetting or -as [List of string]

Adds a setting to the SSoT.me Project

-removeSetting or -rs [List of string]

REmoves a setting from the SSoT.me Project

-keyFile or -f [string]

The keyfile to use. By default it looks for ~/.ssotme/ssotme.key. (or ~/.ssotme/ssotme.{username}.key)

-addTranspiler [string]

Add a transpiler to for the given account

-deleteTranspiler [string]

Delete the transpiler with the given name

-waitTimeout or -w [int]

The amount of time to wait for the command to continue

-runAs or -ra [string]

Run as this user (look for this user's key file)

-preserveZFS or -rz [bool]

Determines if the input should be preserved.

-checkResults or -cr [bool]

Checks the result of a build linking up input and output files of the transpiles. Creates a SPXML file in the DSPXml folder of the project.

-createDocs or -cd [bool]

Creates documentation based on a DSPXml file created with the -checkResults flag.

-execute or -exec [string]

Executes the given command as a ProcessInfo.Start

-includeDisabled or -id [bool]

Include disabled tools in th ebuild

-projectName or -name [string]

Name of the project (optional parameter to the init command)

-transpilerGroup or -tg [string]

Name of a group to put a transpiler in within a specific folder

-setAccountAPIKey or -api [string]

Add an account api key

-authenticate or -auth [bool]

Launch the SSoT.me website in order to authenticate (and/or register), and then to link that user to your ssotme CLI.

-repoUrl [string]

Override the default URL specified by the seed repository

-betaRepo [bool]

Use the beta repository for this seed?

-skipBuild [bool]

Skips the build part of cloning a Seed repository

-info [bool]

Show configured settings

-version or -v [bool]

Show CLI version

-dryRun or -dr [bool]

Dry run of a buid

-targetUrl or -g [string]

TargetUrl of the tool bing invoked

-listToolUrls or -lt [bool]

List all custom tool urls defined for this project

-viewToolUrl or -vt [string]

View the url defined for the specified tool for this project

-setToolUrl or -st [string]

Set a tool's URL to a custom endpoint for this project

-removeToolUrl or -rt [string]

Removes a custom tool URL for this project, setting it back to the default value.
